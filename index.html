    Web 1.0 reads, and gets information.
    Web 2.0 reads, wrires and creates. So, users joined social platforms, and these platforms got big because of this created content.
    Web 3.0 reads writes and owns. So, builders and creators can now own a piece of their own community, through NFTs, tokens etc.


    To sum up the differences I provided a table:
    Web 1.0 Web 2.0 Web 3.0
    Owned content Shared content Consolidated content
    Typically read-only Strongly read-write Read-write-interact
    Visual/interactive Web Programmable Web Linked data Web
    Web page Web service endpoint Data space
    Banner advertising Interactive advertising Behavioral advertising
    Active 1989-2005 Active 1999-2012 Active 2006-ongoing

    Quewstion 2:


    Use-Case1: Content
    Web 1.0 the content of it is mainly read only(Static, Decentralized).For an example a business website. The website built manily from basic HTML pages with text and images, providing information about the company and its products or services. Visitors of the web-site would be able to read only the content. The content is programmer-generated.
    Web 2.0 the content could be read and written(Dynamic, Consolidated).For an example Facebook. The social media platform the best example of Web 2.0 content. it allows users to write: upload post updates, share photos and videos. And read: connect to friends, pages. The site is highly interactive, with users able to like and comment on each other's posts, join groups, and participate in discussions. The content on is user-generated.
    Web 3.0 the content can be read, written and interacted(Semantic, Democratize). Blockchain-based webs are the best example of the web 3.0 content. Like Steemit for an example. Uusers can create and share content, yet it is designed to be decentralized, (no central authority can controle the content). Users are incentivized to create high-quality content by earning cryptocurrency rewards for their contributions. The platform uses blockchain technology.
